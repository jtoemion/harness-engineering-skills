# Development Guide

## lede

Universal agent enforcement ecosystem for AI coding agents. Prerequisites: Python 3.10+, Node.js 18+. Single start: `python harness/runtime/harness.py boot`.

## Prerequisites

| Tool | Required | Check |
|------|----------|-------|
| Python | 3.10+ | `python --version` |
| Node.js | 18+ | `node --version` |
| npm | any | `npm --version` |
| Git | optional | `git --version` |
| pydantic | 2.x | `python -c "import pydantic"` |

Install Python dependencies:
```bash
pip install pydantic pyyaml
```

Install flow-watcher dependencies:
```bash
cd harness/runtime/flow-watcher && npm install
```
The nli-deberta-v3-base model (~250MB) downloads automatically on first run.

## Gotchas

| Gotcha | What it means | Don't do this |
|--------|---------------|---------------|
| conductor.py imports old `classify_with_qwen` | bridge.py no longer exports it (renamed to WatchBridge.watch()) | Expect LLM routing to work — it falls back to YAML keyword matching |
| harness.py imports `QwenBridge` | bridge.py now has `WatchBridge` instead | Expect bridge to warm up on boot — import fails silently, bridge skipped |
| skill-router.yaml has hardcoded paths | `skills_root: "C:\\Users\\jtoem\\...` — Windows-specific, personal paths | Copy the repo and use as-is — these paths won't match your machine |
| AGENTS.md is gitignored | Contains personal local paths (C:\Users\jtoem\) | Commit AGENTS.md — it has your secrets and machine paths |
| .memory/ controls mode detection | Presence of `.memory/` in project root = FULL mode | Create `.memory/` if you want FULL mode enforcement |
| __pycache__/ is gitignored | Python bytecode cache excluded | Trust __pycache__ to be in the repo |
| flow-watcher .cache/ is gitignored | ONNX model cache excluded (~500MB xsmall + base) | Re-downloads models on each machine |
| conductor.py reads `ROUTER_YAML` as `ROUTES` | But skill-router.yaml uses `routing:` key | Rename key in skill-router.yaml or conductor.py breaks |
| session close is mandatory in FULL mode | pre-commit hook blocks commits if state != CLOSED | Skip close and expect to commit — hook will block it |

## Environment Setup

No `.env` required for the harness runtime itself. The browser-agent subskill uses `.env` files (gitignored).

```bash
# Verify Python dependencies
python -c "import pydantic, yaml; print('OK')"

# Verify Node.js / npm
node --version && npm --version

# Verify flow-watcher can start (first run downloads model ~250MB)
cd harness/runtime/flow-watcher && node flow-watcher.js
# Type: {"input": "test"} then Enter
# Should return: {"alerts":[],"gate":"PASS","reason":"nli-watch"}
```

```bash
python browser-agent/_tools/self_test.py

# Flow-watcher test (manual)
cd harness/runtime/flow-watcher
node -e "
const fw = require('./flow-watcher.js');
// Note: interactive — use the test harness or pipe input

```

## running

```bash
# Boot (FULL mode — creates .harness-state.json)
python harness/runtime/harness.py boot

# Pre-task gate
python harness/runtime/harness.py gate --phase pre-task --input "fix the login bug"

# Mark verification complete
python harness/runtime/harness.py verify-done

# Run checkpoint
python harness/runtime/harness.py checkpoint --task "Fix login bug"

# Pre-complete gate
python harness/runtime/harness.py gate --phase pre-complete

# Session close (12-step, resumable)
python harness/runtime/harness.py close

# Resume interrupted close
python harness/runtime/harness.py close --resume

# Print status
python harness/runtime/harness.py status

# List active mistakes
python harness/runtime/harness.py mistakes list
```

## Git Hooks

```bash
# Install pre-commit hook (blocks commit if session not closed)
python harness/runtime/hooks/install_hooks.py
```

## Building / Deploying

No build step — this is a Python + JavaScript runtime, not a compiled project. Deploy by cloning the repo and running `npm install` in the flow-watcher directory.

For use as a submodule in other projects:
```bash
git submodule add https://github.com/jtoemion/harness-engineering-skills.git skills
```

## File Structure Notes

- **harness/runtime/** — All live enforcement code. Imports use relative paths from runtime dir.
- **skill-router.yaml** — Single source of truth for all skill routing. Update here, not in individual skills.
- **SKILLS.yaml** — Address map with categories. Mostly informational for Antigravity config.
- **superpowers/** — 35 imported skills. Do not modify directly (upstream from github.com/obra/superpowers).
- **browser-agent/** — Independent sub-skill. Can be used separately from harness.
- **.docs/** — Auto-generated by `python enforce.py start --mode initial_scan`. Do not edit manually.
