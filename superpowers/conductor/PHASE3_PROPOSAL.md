# Phase 3 — Pipeline Proposal

Present this before executing. Then stop and wait.

## Format

```
⚡ CONDUCTOR — WORKFLOW PROPOSAL
Request  : [one-line summary]
Intent   : [primary goal]
Tier     : [Simple / Moderate / Complex / Ultrathink]
Skills   : [comma-separated list]
Sessions : [1 / 2 / N — how many sessions needed]

PIPELINE — [N] stages ([M] this session, [R] deferred)

STAGE 1: [skill-name] [ANTIGRAVITY/SUPERPOWERS]
  Goal   : [what this achieves]
  Scope  : [files this stage may touch]
  Verify : [how to confirm completion]
  Cost   : [Small / Medium / Large]

STAGE 2: [skill-name]
  ...

--- SESSION BOUNDARY (if applicable) ---

STAGE 4+: deferred to next session
  Carry  : [artifacts/decisions to pass forward]

Reply GO / ADJUST [N] / SKIP [N] / SWAP [N]
```

---

## Phase 4 — Await Approval

```
- Present the pipeline. Stop completely.
- Accept only: GO / ADJUST / SKIP / SWAP / question
- On modification → update, re-present
- Do NOT begin execution until explicit GO
```
